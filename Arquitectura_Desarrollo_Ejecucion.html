Arquitectura de Desarrollo y Ejecución del Sistema

Este documento detalla la infraestructura tecnológica y operacional del sistema, abarcando desde el ciclo de desarrollo hasta el entorno de ejecución final del usuario.

1. Entorno de Desarrollo (Desarrollo)

El ciclo de vida del software se gestiona bajo un ecosistema de herramientas de vanguardia que garantizan la eficiencia y la calidad del código antes de su despliegue.

[ FIGURA 1: DIAGRAMA DE ENTORNO DE DESARROLLO - PEGAR IMAGEN AQUÍ ]

Figura 1: Esquema de herramientas en el entorno de desarrollo.

1.1 Herramientas y Frameworks de Construcción

  • Antigravity AI (IDE & Agent): Núcleo inteligente utilizado para la generación, auditoría y optimización del código fuente en tiempo real.
  • Git & GitHub: Sistema de control de versiones distribuido que aloja los repositorios centrales del proyecto (Cliente y Servidor).
  • Postman & ngrok: Herramientas críticas para la validación de endpoints API y la exposición segura de servicios locales para pruebas remotas.
  • Docker: Orquestación de contenedores para asegurar que el entorno de desarrollo sea idéntico al de ejecución.

2. Entorno de Ejecución (Ejecución)

La arquitectura de ejecución está diseñada para ser escalable, segura y reactiva, permitiendo una interacción fluida entre el usuario y los datos.

[ FIGURA 2: FLUJO DE EJECUCIÓN Y PERSISTENCIA - PEGAR IMAGEN AQUÍ ]

Figura 2: Diagrama de flujo de datos y persistencia en el servidor.

2.1 El Lado del Cliente (Frontend)

El componente Cliente se encarga de la interfaz de usuario y la experiencia visual. Utiliza tecnologías como React y Socket.io para mantener una comunicación bidireccional y asíncrona.

2.2 El Lado del Servidor (Backend)

El Servidor actúa como el motor de lógica de negocio, procesando peticiones a través de túneles seguros. Entre sus funciones principales se encuentran:

  • Gestión de métodos HTTP estándar: GET, POST, PUT, DELETE.
  • Validación de datos de entrada mediante validadores estrictos (Zod/FluentValidation).
  • Implementación de DTOs (Data Transfer Objects) para el transporte seguro de información.

2.3 Servicios de Terceros y Persistencia

El sistema se integra con servicios externos como Libélula y sistemas de mensajería/correo para notificaciones y procesos administrativos.

2.4 Capa de Persistencia de Datos

La base de datos PostgreSQL gestiona toda la información transaccional bajo contenedores Docker. La interacción se realiza mediante un ORM (Object-Relational Mapping), el cual traduce la lógica de objetos del programador en consultas SQL optimizadas.

3. Flujo de Comunicación y Protocolos

La comunicación se basa en el intercambio de HTTP JSON. El flujo típico inicia en el Navegador del usuario, viaja a través de un túnel de ngrok, es procesado por el Servidor y finalmente interactúa con la Base de Datos para devolver una respuesta en milisegundos al Cliente final.

Nota informativa: Toda la arquitectura está respaldada por el protocolo MCP para la integración continua de servicios inteligentes durante la operación.
